# Основные команды Git

---

## Инициализация и получение репозитория

- `git init`  
  Инициализирует новый локальный git-репозиторий в текущей папке. Использовать если создаёте репозиторий с нуля.

- `git clone <url>`  
  Клонирует удалённый репозиторий на локальный компьютер. Рекомендуется сразу копировать репозиторий с [GitHub](https://github.com)

---

## Проверка состояния и истории

- `git status`  
  Показывает текущее состояние репозитория (изменённые, добавленные и неотслеживаемые файлы).

- `git log`  
  Показывает историю коммитов.

- `git log --oneline --graph`  
  Компактная история коммитов с визуализацией веток.

- `git diff`  
  Показывает изменения в файлах, которые ещё не добавлены в индекс.

- `git diff --staged`  
  Показывает изменения, добавленные в индекс (staging area).

---

## Работа с файлами

- `git add <file>`  
  Добавляет файл в индекс (staging area).

- `git add .` или `git add -A`  
  Добавляет все изменения в текущей директории.

---

## Коммиты

- `git commit -m "message"`  
  Создаёт коммит с сообщением.

- `git commit --amend`  
  Изменяет последний коммит (сообщение или состав файлов).

---

## Работа с ветками

- `git branch`  
  Показывает список веток.

- `git branch <name>`  
  Создаёт новую ветку.

- `git checkout <branch>`  
  Переключается на ветку.

- `git checkout -b <branch>`  
  Создаёт новую ветку и сразу переключается на неё.

- `git switch <branch>`  
  Современная альтернатива `git checkout`.

- `git branch -d <branch>`  
  Удаляет ветку.

---

## Слияние и история

- `git merge <branch>`  
  Сливает указанную ветку в текущую.

- `git rebase <branch>`  
  Переносит коммиты текущей ветки поверх другой.

---

## Работа с удалённым репозиторием

- `git remote -v`  
  Показывает список удалённых репозиториев.

- `git pull`  
  Загружает и сразу сливает изменения (`fetch + merge`).

- `git push`  
  Отправляет коммиты в удалённый репозиторий.

- `git push -u origin <branch>`  
  Отправляет ветку и связывает её с удалённой.

---

## Работа с конфликтами

- `git merge --abort`  
  Отменяет merge при возникновении конфликта.

- `git rebase --abort`  
  Отменяет rebase.

- `git rebase --continue`  
  Продолжает rebase после разрешения конфликтов.

---

## Полезные дополнительные команды

- `git stash`  
  Временно сохраняет незакоммиченные изменения.

- `git stash pop`  
  Возвращает сохранённые изменения.

- `git blame <file>`  
  Показывает, кто и когда изменял строки файла.

- `git show <commit>`  
  Показывает содержимое конкретного коммита.

---

## Минимальный набор команд

- `git clone`
- `git status`
- `git add`
- `git commit`
- `git pull`
- `git push`
- `git branch`
- `git checkout` / `git switch`
- `git merge`
- `git log`
